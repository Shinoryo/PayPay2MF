# 設定ファイル YAML スキーマ仕様書

## 概要

本ドキュメントは、PayPay→MoneyForward 自動登録ツールの設定ファイル（`config.yml`）の YAML スキーマ仕様を定義する。

## ファイル情報

- **ファイル名**：`config.yml`
- **配置場所**：任意。起動時は `--config` > 環境変数 `PAYPAY2MF_CONFIG` > カレントディレクトリ > モジュール同居 の順に探索する
- **フォーマット**：YAML 1.2
- **文字コード**：UTF-8（BOM なし）

## バリデーション方針

- ツール起動時に `config.yml` の存在チェックおよびスキーマバリデーションを行う。
- `config.yml` の探索順は `--config` > 環境変数 `PAYPAY2MF_CONFIG` > カレントディレクトリ > モジュール同居とする。
- `config.yml` 自体にもファイルパスを要求し、存在してもディレクトリは受け付けない。
- `config.yml` のルート要素は object である必要があり、list や scalar は受け付けない。
- 必須項目が欠落、空文字・空白のみ、または型不正の場合は起動を中止し、ユーザーに具体的なエラーメッセージを表示する。
- 必須項目は 3 項目（`dry_run` / `input_csv` / `mf_account`）。
- 任意項目が未指定または `null` の場合は指定のデフォルト値を使用する。
- object セクションは期待するキー構造のみを受け付け、未知キーや型不正は起動時エラーとする。

## スキーマ定義

### ルートレベル

```yaml
# 型表記凡例
# string        : 文字列
# boolean       : true / false
# integer       : 整数
# list[T]       : T 型の要素を持つリスト
# object        : キーと値のマッピング
# null           : null（YAML の null キーワード）
# T | null      : T 型または null
```

### 1. dry_run

| 項目 | 内容 |
| ------ | ------ |
| 型 | boolean |
| 必須 | **必須**（未指定で起動エラー） |
| デフォルト | なし（必ず明示指定すること） |
| 有効値 | `true` / `false` |

#### 説明

`true` の場合、ブラウザは起動せず CSV 診断結果のみを出力するドライランモードで動作する。
`false` の場合、実際に MF に登録を行う（本番実行）。
意図せず本番実行することを防ぐため、デフォルトを設けず必ず明示指定とする。
この設定は本体アプリ専用であり、`paypay2mf-firestore-backfill` の dry-run は
CLI 引数 `--dry-run` で別に指定する。

#### バリデーションエラー例

```text
[ERROR] dry_run が設定されていません。true または false を config.yml に記載してください。
```

### 2. input_csv

| 項目 | 内容 |
| ------ | ------ |
| 型 | string |
| 必須 | **必須**（未指定で起動エラー） |
| デフォルト | なし（必ず明示指定すること） |
| 参考値 | `C:\Users\yourname\Downloads\paypay_history.csv` |

#### 説明

処理対象の CSV ファイルの絶対パスまたは
`config.yml` の配置ディレクトリ基準の相対パス。実行時のカレントディレクトリではなく、常に `config.yml` が置かれた場所を基準に解決する。起動時に以下を検証する。

- 空文字・空白のみではないこと。
- ファイルが存在すること。
- ディレクトリではなくファイルであること。
- 拡張子が `.csv` であること。
- 相対パス指定時は `config.yml` の配置ディレクトリ基準で解決すること。

#### バリデーションエラー例

```text
[ERROR] input_csv が設定されていません。config.yml に記載してください。
[ERROR] input_csv のファイルが存在しません: C:\path\to\file.csv
```

### 3. mf_account

| 項目 | 内容 |
| ------ | ------ |
| 型 | string |
| 必須 | **必須**（未指定で起動エラー） |
| デフォルト | なし（必ず明示指定すること） |
| 参考値 | `PayPay` |

#### 説明

MF の手入力フォームで選択する口座名。
口座プルダウンに `PayPay (439,670円)` のような残高サフィックスが表示される環境でも、
設定値は末尾の `(<金額>円)` を除去した名称で照合するため、通常は口座名本体
（例: `PayPay`）を指定する。正規化後に同名候補が複数ある場合は曖昧一致としてエラーにする。
空文字・空白のみは無効とし、起動時にエラーとする。

#### バリデーションエラー例

```text
[ERROR] mf_account が設定されていません。config.yml に記載してください。
```

### 6. mapping_rules

| 項目 | 内容 |
| ------ | ------ |
| 型 | list[object] |
| 必須 | 任意（省略時は全取引が「未分類」となる） |
| デフォルト | 空リスト `[]` |

#### 説明

取引先名からカテゴリを決定するためのルール一覧。
適用順は、まず `priority` の降順（数値が大きいほど優先）で評価し、
同一 `priority` の場合は `direction` 指定ルール（`income` / `expense`）を
`any` より優先する。

#### 各ルールオブジェクトのスキーマ

| フィールド | 型 | 必須 | デフォルト | 説明 |
| ----------- | --- | ------ | ----------- | ------ |
| keyword | string | 必須 | — | マッチングに使うキーワード（入力値をそのまま保持） |
| category | string | 必須 | — | MF に登録するカテゴリ名（前後空白は除去して保持） |
| match_mode | string | 任意 | `"contains"` | `"contains"`（部分一致） / `"starts_with"`（前方一致） / `"regex"`（正規表現） |
| direction | string | 任意 | `"any"` | `"income"` / `"expense"` / `"any"`（前後空白は除去して判定・保持） |
| priority | integer | 任意 | `0` | 数値が大きいほど先に評価される |

#### バリデーション

- `keyword` または `category` が空文字の場合はエラー。
- `match_mode` が上記3値以外の場合はエラー。
- `direction` は `"income"` / `"expense"` / `"any"` のみ許可する。
- `keyword` は前後空白を保持する（空白のみ文字列はエラー）。
- `category` / `direction` は前後空白を除去して正規化する。
- `direction` を指定する場合、空文字・空白のみはエラー。
- `priority` は 0 以上の整数のみ許可する。文字列、真偽値、負数、null はエラー。
- 未定義の追加キーは許可しない。

#### direction と CSV 入出金列の対応

- `direction: "income"` は、CSV の `入金金額（円）` が正で `出金金額（円）` が 0 の行に適用する。
- `direction: "expense"` は、CSV の `出金金額（円）` が正で `入金金額（円）` が 0 の行に適用する。
- `direction: "any"`（または未指定）は、上記どちらの方向にも適用する。

#### 初期推奨ルール（サンプル）

| keyword | category |
| --------- | ---------- |
| セブン | 食料品 |
| ファミリーマート | 食料品 |
| イオンシネマ | 映画・音楽・ゲーム |
| Yahoo!ショッピング | その他日用品 |
| Google - GOOGLE PLAY | 情報サービス |
| モス | 外食 |
| キャンドゥ | 日用品 |
| つきじ海賓 | 外食 |
| MYB | 雑費 |
| giftee | その他入金 |

### 7. exclude_prefixes

| 項目 | 内容 |
| ------ | ------ |
| 型 | list[string] |
| 必須 | 任意 |
| デフォルト | `["PPCD_A_"]` |

#### 説明

取引番号がこのリスト内のいずれかのプレフィックスで始まる場合、その行を処理対象外（除外）とする。

#### バリデーション

- `exclude_prefixes` は list のみ許可する。単一の string 指定はエラー。
- 各要素は空文字・空白のみではない文字列である必要がある。

### 8. duplicate_detection

| 項目 | 内容 |
| ------ | ------ |
| 型 | object |
| 必須 | 任意 |
| デフォルト | 下記参照 |

#### 子フィールドのスキーマ

| フィールド | 型 | 必須 | デフォルト | 説明 |
| ----------- | --- | ------ | ----------- | ------ |
| backend | string | 任意 | `"local"` | `"local"`：JSONファイルで重複履歴管理 / `"gcloud"`：Google Cloud（Firestore等）で管理。利用する場合は `gcloud_credentials_path` を指定した上で明示的に設定する |
| database_id | string \| null | 任意 | `"(default)"` | Firestore のデータベース ID。`backend: "gcloud"` で複数 DB を運用する場合に対象 DB（例: `"paypay2mf"`）を明示指定する |

#### 動作仕様

- 重複判定は行単位指紋（sha256）で行う。
- 指紋入力項目は `取引日（CSV文字列）`、`取引内容`、`取引先`、
  `出金金額（正規化）`、`入金金額（正規化）`、`取引方法`、`支払い区分`、`利用者`。
- `backend: "gcloud"` の場合、Firestore の document id には行指紋を使用する。
- 同一 `transaction_id` でも行指紋が異なる場合は別取引として扱う。
- `backend: "local"` の場合、処理済み履歴は実行中はメモリに保持し、登録ループ終了時に
  `processed.json` へ 1 回だけ原子的に保存する。

#### Firestore の query / index 方針

- 現在の本体アプリは document id に設定した `row_fingerprint` を直接参照して重複判定する。
- そのため、通常運用の重複判定だけであれば追加の複合 index は不要。
- `date_bucket` は backfill、運用調査、将来の一覧取得を見据えた検索補助項目として payload に保持する。
- `date_bucket` や `amount` などの payload フィールドで
  where/order_by を組み合わせるクエリを追加する場合は、
  実際のクエリ形に応じて Firestore 側の複合 index を別途作成する。
- `duplicate_detection.database_id` を切り替える場合、index の管理対象もそのデータベースごとに分かれる。

#### backfill CLI での扱い

- `paypay2mf-firestore-backfill` はこのセクションのうち
  `backend: "gcloud"` と `database_id` を参照する。
- backfill CLI では `backend: "gcloud"` が必須で、`"local"` のままでは起動しない。
- backfill CLI では `database_id` に対して本体アプリと同じ型検証を適用する。
- backfill CLI は `input_csv` や `mf_account` を参照しない。

#### バリデーション

- セクション自体は object または `null` のみ許可する。
- `backend` は `"local"` または `"gcloud"` のみ許可する。
- `database_id` は string または `null` のみ許可し、空文字はエラー。
- 未定義の追加キーは許可しない。

#### タイムゾーンに関する注意

- 重複判定に使う取引日は PayPay CSV の文字列値をそのまま使い、
  設定でタイムゾーン変換や再フォーマットは行わない。
- `processed.json` の内容が破損または不正スキーマで退避ファイルを作成する場合、
  退避ファイル名 `processed.corrupted_YYYYMMDD_HHMMSS_ffffff.json` の
  タイムスタンプは UTC を使う。

### 9. parser

| 項目 | 内容 |
| ------ | ------ |
| 型 | object |
| 必須 | 任意 |
| デフォルト | 下記参照 |

#### 子フィールドのスキーマ

| フィールド | 型 | 必須 | デフォルト | 説明 |
| ----------- | --- | ------ | ----------- | ------ |
| encoding_priority | list[string] | 任意 | `["utf-8", "shift_jis"]` | CSV読み込み時に試す文字コード順 |
| date_formats | list[string] | 任意 | `["%Y/%m/%d %H:%M:%S"]` | 日付解析フォーマットの候補（先頭から順に試す） |

#### バリデーション

- セクション自体は object または `null` のみ許可する。
- `encoding_priority` と `date_formats` は list のみ許可する。
- `encoding_priority` と `date_formats` は空 list を許可しない。
- `encoding_priority` と `date_formats` の各要素は空文字・空白のみではない文字列である必要がある。
- 要素エラー時は `parser.encoding_priority[0]` のように index 付きでエラーを返す。
- 未定義の追加キーは許可しない。

#### パース失敗時の動作

- 行単位の値不正や必須列欠損は `parse_error_yyyyMMdd_HHmmss.csv` に記録し、他の正常行の処理は継続する。
- エンコーディング検出失敗やファイル読み込み失敗など、ファイル全体を処理できない場合のみ起動エラーとする。
- `error_type` には `missing_column`、`invalid_date`、`invalid_value`、
  `conversion_error` のいずれかを出力する。

### 10. log_settings

| 項目 | 内容 |
| ------ | ------ |
| 型 | object |
| 必須 | 任意 |
| デフォルト | 下記参照 |

#### 子フィールドのスキーマ

| フィールド | 型 | 必須 | デフォルト | 説明 |
| ----------- | --- | ------ | ----------- | ------ |
| logs_dir | string \| null | 任意 | `config.yml` と同じディレクトリ配下の `logs` | ログ／スクリーンショット／エラーCSVの保存先。相対パス指定時は `config.yml` の配置ディレクトリ基準で解決し、null または未指定でデフォルトを使用 |
| max_log_count | integer \| null | 任意 | `null`（無制限） | 保持するログファイル数の上限。超過した場合は古いものから削除 |
| max_total_log_size_mb | integer \| null | 任意 | `null`（無制限） | ログの合計サイズ上限（MB）。超過した場合は古いものから削除 |

#### logs_dir のバリデーション

- `logs_dir` は string または `null` のみ許可する。
- 相対パス指定時は `config.yml` の配置ディレクトリ基準で解決する。
- ディレクトリが存在しない場合は起動時に自動作成を試みる（失敗した場合はエラー）。
- セクション自体は object または `null` のみ許可する。
- `max_log_count` と `max_total_log_size_mb` は `null` または 0 以上の整数のみ許可する。文字列、真偽値、負数はエラー。
- 未定義の追加キーは許可しない。
- `max_log_count: 0` を指定した場合は、過去ログを削除対象にしつつ、現在の実行で生成した app ログは保持する。

### 11. advanced

| 項目 | 内容 |
| ------ | ------ |
| 型 | object |
| 必須 | 任意 |
| デフォルト | 下記参照 |

#### 子フィールドのスキーマ

| フィールド | 型 | 必須 | デフォルト | 説明 |
| ----------- | --- | ------ | ----------- | ------ |
| screenshot_on_error | boolean | 任意 | `false` | true を明示した場合のみ、Selenium driver が利用可能なエラー時にスクリーンショットを保存するか |
| mf_categories_path | string \| null | 任意 | `null`（同梱 `mf_categories.yml` を使用） | Money Forward の中項目→大項目カテゴリ対応表 YAML の差し替えパス |

#### セキュリティ上の注意

- セクション自体は object または `null` のみ許可する。
- `screenshot_on_error` は boolean のみ許可する。
- `mf_categories_path` は string または `null` のみ許可する。数値や list はエラー。
- `logs_dir` 配下のログ、CSV、PNG は機微情報を含む可能性があるため、ローカル保管とし、共有フォルダやクラウド同期先に置かない。
- `error_*.csv` は `failure_index` と `error_message` の最小列のみを出力する。
- `parse_error_*.csv` は `row_index` と `error_type` と `error_message` の最小列のみを出力する。
- スクリーンショットは既定では保存されない。必要時のみ `advanced.screenshot_on_error: true` を明示する。
- Selenium driver が未初期化の状態では、`advanced.screenshot_on_error: true` でもスクリーンショットは保存されず、保存済みログも出力しない。
- `mf_categories_path` は相対パス・絶対パスのどちらでも指定できる。相対パスは `config.yml` の配置ディレクトリ基準で解決する。
- `mf_categories_path` を指定した場合、起動時にファイル存在確認を行い、ディレクトリ指定は拒否する。
- 未指定または `null` の場合は同梱 `mf_categories.yml` を使用する。
- 未定義の追加キーは許可しない。

### 12. gcloud_credentials_path

| 項目 | 内容 |
| ------ | ------ |
| 型 | string \| null |
| 必須 | 任意 |
| デフォルト | `null`（未設定） |
| 参考値 | `C:\Users\yourname\gcloud-service-account.json` |

#### 説明

`duplicate_detection.backend: "gcloud"` の場合に使用する
Google CloudサービスアカウントのJSONキーファイルの絶対パス、または
`config.yml` の配置ディレクトリ基準の相対パス。
`null` または未指定でも、この項目だけで
`duplicate_detection.backend` の設定値は自動変更されない。
backend が `"local"` のままなら、この項目を設定していても利用されない。
`duplicate_detection.backend` を `"gcloud"` に明示した場合は、`gcloud_credentials_path` の指定が必要。
`paypay2mf-firestore-backfill` でも同じパス解決とファイル検証を行い、
backfill 実行時はこの項目が必須となる。

#### バリデーション

- `gcloud_credentials_path` は string または `null` のみ許可する。
- `duplicate_detection.backend: "gcloud"` の場合は、空文字ではなく実在するファイルパスを指定する。
- ディレクトリではなくファイルであること。

> **運用メモ**：`paypay2mf-firestore-backfill` は `date_bucket` を補完する補助CLIであり、
> 本体アプリの重複判定キーは row_fingerprint（document id）を使用する。
> **セキュリティノート**：JSONキーファイルは安全なローカルパスに保存し、バージョン管理システム（Git等）にコミットしないこと。

#### バリデーションエラー例

```text
[ERROR] gcloud_credentials_path のファイルが存在しません: C:\path\to\key.json
[ERROR] gcloud_credentials_path にはファイルを指定してください: C:\path\to\key.json
[ERROR] duplicate_detection.backend: "gcloud" の場合は gcloud_credentials_path の指定が必要です。
```

```yaml
# ============================================================

# ============================================================

# ------ 必須設定（3項目。未記載で起動エラー） ------

dry_run: true   # true: ドライラン（書き込みなし） / false: 本番実行
input_csv: "C:\\Users\\yourname\\Downloads\\paypay_history.csv"
mf_account: "PayPay"

# ------ 任意設定 ------

mapping_rules:
  - keyword: "セブン"
    category: "食料品"
    match_mode: "contains"
    direction: "expense"
    priority: 100
  - keyword: "ファミリーマート"
    category: "食料品"
    match_mode: "contains"
    direction: "expense"
    priority: 100
  - keyword: "イオンシネマ"
    category: "映画・音楽・ゲーム"
    match_mode: "contains"
    priority: 100
  - keyword: "Yahoo!ショッピング"
    category: "その他日用品"
    match_mode: "contains"
    priority: 100
  - keyword: "Google - GOOGLE PLAY"
    category: "情報サービス"
    match_mode: "contains"
    priority: 100
  - keyword: "モス"
    category: "外食"
    match_mode: "contains"
    priority: 100
  - keyword: "キャンドゥ"
    category: "日用品"
    match_mode: "contains"
    priority: 100
  - keyword: "つきじ海賓"
    category: "外食"
    match_mode: "contains"
    priority: 100
  - keyword: "MYB"
    category: "雑費"
    match_mode: "contains"
    priority: 100
  - keyword: "giftee"
    category: "その他入金"
    match_mode: "contains"
    direction: "income"
    priority: 100

exclude_prefixes:
  - "PPCD_A_"

duplicate_detection:
  backend: "local"
  # database_id: "paypay2mf"  # Firestore DB ID。未指定時は "(default)"

    - "utf-8"

    - "%Y/%m/%d %H:%M:%S"

  logs_dir: null            # null → デフォルトは config.yml と同じディレクトリ配下の logs

  max_total_log_size_mb: null  # null = 無制限

advanced:
  screenshot_on_error: false
  mf_categories_path: null
```
