# テストケース詳細手順書

## 概要

本ドキュメントは、PayPay→MoneyForward 自動登録ツールの
受入テストケースおよびチェックリストを定義する。
pytest による自動テストと手動確認を対応付けて管理し、
テストは「単体テスト」「integration_flow」「ui_contract」
「smoke_test」「手動 E2E」の5区分で運用する。

## 自動化区分と実行コマンド

| 区分 | 目的 | pytest marker | 代表コマンド | 主な対象 |
| ------ | ------ | ------ | ------ | ------ |
| 単体テスト | モジュール単位の純粋ロジック検証 | なし | `python -m pytest -q` | `test_config_loader.py`, `test_csv_parser.py`, `test_filter.py`, `test_duplicate_detector.py`, `test_log_manager.py`, `test_models.py`, `test_mf_category_map.py`, `test_firestore_backfill.py` |
| integration_flow | main フローのオーケストレーションと副作用 | `integration_flow` | `python -m pytest -q -m integration_flow` | `test_main.py` |
| ui_contract | Fake を使った UI 契約検証 | `ui_contract` | `python -m pytest -q -m ui_contract` | `test_mf_page.py`, `test_mf_registrar.py` |
| smoke_test | 実ブラウザでの明示実行スモーク | `smoke_test` | `python -m pytest -q -m smoke_test tests/test_mf_smoke.py` | `test_mf_smoke.py` |
| 手動 E2E | 実アカウントへの書き込みを伴う受入確認 | 該当なし | 手順書ベースで実施 | TC-07-01, TC-07-02 |

## テスト環境前提条件

| 項目 | 内容 |
| ------ | ------ |
| OS | Windows 11 |
| ブラウザ | Google Chrome（最新版） |
| Python | 3.11 以上 |
| ライブラリ | pyproject.toml に定義された依存関係一式（Playwright, psutil, PyYAML など）がインストール済み |

## TC-04：重複検知テスト（単体／結合）

**主な対応 pytest**：`tests/test_duplicate_detector.py`

> 注記: GCloud backend は optional dependency のため、標準自動テストでは Firestore client をモックした単体テストまでを対象とする。実サービス接続と権限確認は手動確認で扱う。

### TC-04-01：取引番号による重複検知

- [ ] 取引番号が空の行を含むテストCSVを用意する。

- [ ] `dry_run: true` では Firestore への書き込みが抑止されることを確認する。

## タイムゾーンに関する補足

- PayPay CSV の `取引日` は記載値をそのまま扱い、テストでも
  タイムゾーン変換を前提にしない。
- `processed.json` 破損時の退避ファイル名だけは UTC のタイムスタンプを使う。
- ログや通常の出力ファイル名は実行環境のローカル時刻を前提に確認する。

## TC-07：E2E テスト（本番実行）

**目的**：フル自動で PayPay CSV から MF に取引が正しく登録されることを確認する。

**自動化区分**：TC-07-00 は smoke_test、TC-07-00A は単体 + 手動、TC-07-01 / TC-07-02 は手動 E2E

> ⚠️ 本テストは実際の MF アカウントに書き込みを行う。テスト前に MF 側でテスト確認しやすい状態（過去の取引が少ない期間等）を準備することを推奨する。

### TC-07 前提条件

- Chrome を完全終了していること。
- Chrome プロファイルで MF にログイン済みであること（Cookieが有効）。
- `config.yml` の `dry_run: false` であること。

### TC-07-00：Playwright スモークテスト（手入力モーダル起動確認）

- [ ] Chrome プロファイルが Money Forward ME にログイン済みであることを確認する。
- [ ] PowerShell で `PAYPAY2MF_RUN_SMOKE_TEST=1` を設定する。
- [ ] `PAYPAY2MF_SMOKE_CHROME_USER_DATA_DIR`、
  `PAYPAY2MF_SMOKE_CHROME_PROFILE`、

## 受入基準（全体）

以下の全てを満たすことを受入条件とする。

| 基準 | 内容 |
| ------ | ------ |
| 設定バリデーション | 必須項目欠落時に100%起動エラーが発生すること |
| CSV変換精度 | テストサンプル全行中99%以上が期待どおりに変換されること |
| 除外ルール | PPCD_A_ で始まる取引が出力に含まれないこと（0件） |
| 重複防止 | 同一CSV2回実行で2回目が100%スキップされること |
| ドライラン | dry_run: true 時にブラウザが起動せず CSV 診断結果のみが出力されること |
| E2E登録 | MF に取引が正しく登録されること（ユーザー目視確認） |
| 障害ハンドリング | エラー時にスクショ・ログが保存され中断原因が表示されること |
| 異常系 | 不正行がエラーCSVに出力され処理が継続されること |
