# 要件定義書

## 概要

PayPayが出力する利用明細（CSV）を取り込み、Money Forward ME
（以下、MF）へ自動で記録（登録）するための Windows 向けローカル
CLI ツール。

ユーザー（Shinoryo）自身の PC 上で動作し、MF 認証情報はクラウドに
保存しない方式を採用する。ドライラン（CSV 診断）モードと、
フル自動（MF への Playwright 操作による登録を自動化）を提供する。

通常の利用形態は Python パッケージとしての実行（`paypay2mf`）とし、
必要に応じて PyInstaller ベースの EXE 配布手順を提供する。

## 背景・目的

- **背景**：PayPayの利用明細を目視で確認し、MFに手動入力しているため工数が大きい。
- **目的**：PayPay CSVを選択するだけでMFへの登録を自動化し、入力ミスと作業時間を削減する。
- **成功基準（KPI）**
  - 手動入力工数を70%以上削減。
  - CSV取引の変換精度99%（サンプル100件中99件期待どおり）。
  - 重複登録を発生させない（重複はスキップまたは明示通知）。
  - 「PPCD_A_」で始まる取引（クレジットカード取引）は登録対象外。

## 対象範囲

- **対象データ**：PayPayが出力するCSV（提供されたカラム構成を想定）
- **対象機能**：CSVの読み込み・パース・フィルタ（除外ルール）・カテゴリマッピング・MF用登録の自動実行・ログ／エラー出力
- **実行環境**：Windows 11 上のローカル CLI 実行（Python 実装）。
  Playwright を利用し、必要に応じて PyInstaller ベースの EXE 配布手順を
  提供する。
- **対象ユーザー**：Shinoryo（単独運用）
- **除外**：MF の内部会計ロジックの自動最適化、マルチユーザー運用（初期段階）

## 利用者・関係者

- **主利用者**：Shinoryo（依頼者）
  - 役割：`config.yml` 設定 → ツール実行（CUI） → 結果確認・保守
  - 運用制約：他者データにアクセスさせないこと、ユーザー自身が保守すること
- **外部関係者**：なし（個人運用のため）

### 各機能の詳細

#### F01 CSV取り込み

- Shift_JIS / UTF-8 の文字コードに対応する。
- parser.encoding_priority に従い、順番に文字コードを試みる。
- カンマ区切り・ダブルクォートを正しく処理する。

#### F02 CSVパーシング

- PayPay CSV の全カラム:
  `取引日` / `出金金額（円）` / `入金金額（円）` / `海外出金金額` /
  `通貨` / `変換レート（円）` / `利用国` / `取引内容` / `取引先` /
  `取引方法` / `支払い区分` / `利用者` / `取引番号`

- 金額文字列（例: `"1,280"`）を数値（整数）に変換する。
  `"-"` は `0` として扱う。

- 有効な取引は `出金金額（円）` と `入金金額（円）` のどちらか一方のみが
  正数であること。

- 両方 `0` の行、および両方が正数の行は CSV 解析失敗として扱う。

- 複合支払（例: 「PayPayポイント (73円), クレジット VISA 4575
  (257円)」）は金額を合算して 1 件として扱う。

- 出金・入金の判定: `出金金額（円）> 0` → 出金、
  `入金金額（円）> 0` → 入金。

- 海外取引（`海外出金金額` が `"-"` 以外）は登録金額に
  `出金金額（円）` を使用し、メモに「海外出金金額 + 通貨」を追記する。

- MF の内容（メモ）欄には `取引内容` カラムの値を入力する。

#### F03 除外ルール

- config.yml の exclude_prefixes（初期値: ["PPCD_A_"]）に合致する取引番号の行は処理対象外とする。
- 除外された行は件数サマリーに反映する。

#### F04 重複検知

- 取引番号が存在する場合はそれを最優先キーとして重複判定を行う。
- 取引番号が欠損している場合は「日時＋金額＋取引先」の組み合わせで重複判定する。
- `duplicate_detection.backend: "gcloud"` の場合は、`date_bucket` を用いて
  Firestore 側で候補件数を絞り込み、`datetime` と許容幅を用いて最終判定を
  行う。
- 重複と判定した行はスキップし、ログにその旨を記録する。
- PayPay CSV の取引日時は記載値をそのまま扱い、タイムゾーン変換は行わない。

#### F05 MFへの自動登録

- Playwright を使い、Chrome（ユーザー指定のプロファイル）を自動制御する。
- 既存のChromeプロファイルのセッション（Cookie等）を利用して処理し、MF認証情報をツールに保存しない。
- 自動化操作：MFの手入力フォームへの1件ずつの画面操作（日付・金額・内容・カテゴリ・口座・入出金区別の入力およびボタン押下）。
- 登録する口座は `config.yml` の `mf_account` で指定する。
-- MF の画面セレクターは実装済みのページオブジェクトと定数で管理する。
-- 変更検知は `ui_contract` テスト、`smoke_test`、手動 E2E で確認する。

#### F06 マッピング設定

- config.yml の mapping_rules にキーワードとカテゴリのペアを設定する。
- match_mode（"contains" / "starts_with" / "regex"）と priority（優先度）を設定可能。
- 取引先にキーワードが合致した場合、対応するカテゴリを付与する。
- どのルールにも合致しない場合は未分類とする。

#### F07 実行前チェック

- ツール起動時、Chrome プロセスが稼働していないことを確認する。
- Chrome が稼働中の場合は処理を中断し、ユーザーに Chrome を終了するよう促すメッセージを表示する。

#### F08 ドライランモード

- `dry_run: true` の場合、ブラウザは一切起動しない。
- CSVの読み込み・パース・フィルタ・マッピングのみ実行し、処理件数・除外件数・重複スキップ件数・登録対象件数を標準出力およびログに出力する。
- 実行開始時に「DRY RUN: ブラウザを起動しません。CSV診断のみ実行します。」とログに出力する。

#### F09 ログ出力

- 実行ログ・エラーCSV・スクリーンショットを log_settings.logs_dir に保存する。
- 実行ログには以下の情報を含む：実行日時・処理件数・除外件数・重複スキップ件数・失敗件数。
- CSV 解析に失敗した行は parse_error CSV として別ファイルに出力し、正常な行の処理は継続する。
- MF 登録に失敗した行は従来どおり error CSV として別ファイルに出力する。
- advanced.screenshot_on_error が true の場合、エラー発生時にスクリーンショットを保存する。

#### F10 設定ファイル

- 起動時の config.yml は `--config` > 環境変数
  `PAYPAY2MF_CONFIG` > カレントディレクトリ > モジュール同居 の順に解決する。
- 起動時に config.yml の存在チェックおよびバリデーションを行う。
- 必須項目（`chrome_user_data_dir` / `chrome_profile` / `dry_run` /
  `input_csv` / `mf_account`）が欠落している場合は起動を中止し、
  詳細エラーをユーザーに通知する。
`input_csv` / `log_settings.logs_dir` / `advanced.mf_categories_path`
 / `gcloud_credentials_path` に相対パスを指定した場合は、実行時の
カレントディレクトリではなく config.yml の配置ディレクトリ基準で
解決する。

#### F11 エラーハンドリング

- Playwright の操作失敗時（セレクタ未検出・タイムアウト等）は処理を中断する。
- エラー発生時にスクリーンショットと詳細ログを logs_dir に保存する。
- ログまたは例外メッセージから中断原因を判別できること。
- `processed.json` の破損退避ファイル名に含めるタイムスタンプは UTC とする。

## 非機能要件

| 区分 | 要件 | 優先度 |
| ------ | ------ | -------- |
| セキュリティ | MF認証情報をクラウドに保存しない。Cookie/sessionはローカルの Chrome プロファイルを利用 | 必須 |
| セキュリティ | `duplicate_detection.backend: "gcloud"` 使用時は重複判定に必要な `transaction_id`、または `datetime` / `amount` / `merchant` / `date_bucket` を Firestore に保存する。利用前に保存項目を許容できること | 高 |
| セキュリティ | `logs_dir` 配下のログ、parse_error CSV、error CSV、スクリーンショット、`processed.json` は機微情報を含みうるため、ローカル保存を前提とし、共有フォルダやクラウド同期先に置かないこと | 高 |
| 運用性 | 依頼者単独で保守可能な設計。設定は config.yml の編集で変更できること | 必須 |
| 運用性 | 障害時はログとスクショで原因を特定できること | 必須 |
| 可用性 | オンデマンド実行を想定。常時稼働不要 | 低 |
| コスト | クラウド利用時は無料枠内運用。未使用ならクラウド費用は不要 | 高 |
| 拡張性 | 将来的に別サービスのCSV対応・GUI設定・マッピング学習等を追加可能な設計 | 中 |

## 制約条件

| 制約 | 内容 | 優先度 |
| ------ | ------ | -------- |
| 予算 | 極力コストをかけない（個人運用） | 高 |
| 人的リソース | 運用・保守は依頼者一人 | 必須 |
| セキュリティ | 他者のデータにアクセスさせない | 必須 |

## 設定ファイル仕様（概要）

設定ファイルの詳細は「設定ファイル仕様書.md」を参照。

- **ファイル名・場所**：`config.yml`。起動時は `--config` > 環境変数
  `PAYPAY2MF_CONFIG` > カレントディレクトリ > モジュール同居 の順に探索
- **フォーマット**：YAML
- **必須項目**：`chrome_user_data_dir` / `chrome_profile` / `dry_run` /
  `input_csv` / `mf_account`（未記載で起動エラー）
- **任意項目**：`mapping_rules` / `exclude_prefixes` /
  `duplicate_detection` / `parser` / `log_settings` / `advanced` /
  `gcloud_credentials_path`
- **相対パス方針**：
  - `input_csv` / `log_settings.logs_dir` / `advanced.mf_categories_path`
    / `gcloud_credentials_path` の相対パスは、
    config.yml の配置ディレクトリ基準で解決する。

## 検証方法（受入基準）

テストの詳細手順は「テスト仕様書.md」を参照。

| テスト種別 | 項目 | 合格基準 |
| ----------- | ------ | --------- |
| 単体 | CSVパース（通常支払） | 全カラムを正しく抽出・数値化できること |
| 単体 | CSVパース（複合支払） | 金額が合算されて1件として処理されること |
| 単体 | CSVパース（ポイント入金） | 出金金額なし・入金金額ありの行が正しく処理されること |
| 単体 | 除外ルール（PPCD_A_） | 取引番号が PPCD_A_ で始まる行が出力に含まれないこと |
| 単体 | 重複検知 | 同一取引番号の2回目がスキップされること |
| 単体 | 文字コード | UTF-8 / Shift_JIS 両方で文字化けなく読み込めること |
| 結合 | ドライラン | ブラウザが起動せず、CSV診断結果（処理件数・除外件数・変換内容）のみが出力されること |
| 結合 | ログ出力 | logs_dir にログ・エラーCSV・スクショが正しく保存されること |
| 結合 | 設定バリデーション | 必須項目欠落時に起動エラーが出ること |
| E2E | フル自動実行 | MF に取引が正しく登録されること（ユーザー目視確認） |
| 障害 | DOM変化シミュレーション | セレクタエラー時にスクショ・ログが保存され中断されること |
| 異常系 | 欠損カラム | 不正行が parse_error CSV に出力され他行は処理継続すること |

## 実行フロー（ユーザー操作）

1. `config.yml` を編集し、必須項目を設定する。
2. `dry_run: true` でドライランを実行し、CSVの解析・変換結果を確認する（ブラウザは起動しない）。
3. 問題がなければ Chrome を完全終了したうえで `dry_run: false` に変更して本番実行する。
4. ツールが Chrome を指定の `user_data_dir` / `profile` で起動し、MF手入力フォームへ1件ずつ自動登録する。
5. 実行結果（成功件数・除外件数・重複スキップ件数・失敗件数）をログとエラーCSV／スクリーンショットで出力する。

## 用語説明

- **Playwright**：
  ブラウザ（Chromeなど）をプログラムで自動操作するためのライブラリ。
  本ツールでは、MFの入力画面を自動で操作する用途で利用する。
- **PyInstaller**：
  Pythonで作成したプログラムを、Windows 向け EXE 配布形態として
  まとめるためのツール。
  本ツールでは通常は Python パッケージ版を利用し、必要時のみ
  EXE 配布手順を提供する。
- **PII**（Personally Identifiable Information）：
  個人を特定できる情報のこと。
  本書では、取引金額・取引先などの機微情報を含む個人情報を指す。
